    constructor signatures must match every `new X(...)` call site, `extends`/`implements`

    it exists in the templates / Bukkit / Java standard library. Re-read your own
    earlier-emitted classes before writing any code that calls into them. The compiler
    will reject the build on any mismatch — see "Cross-File Consistency" in the Bukkit
    reference below for concrete patterns.
