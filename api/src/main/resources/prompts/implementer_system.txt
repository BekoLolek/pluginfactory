## Design Principles

- **One responsibility per class.** The main class WIRES things up (registers commands/listeners,
  loads config, creates managers) — it does not implement all the logic itself. Put commands in
  command classes, event handling in listener classes, data in manager/service classes.
- **Store UUIDs, not Player objects.** Players log out; a cached `Player` leaks and goes stale.
  Keep `UUID` and resolve with `Bukkit.getPlayer(uuid)` (null-check) on use.
- **Cache config into fields** in `onEnable()` / a `loadConfig()` method; re-read them on reload.
  Never call `getConfig()` on a hot path.
- **Null-safety on every Bukkit lookup**: `getPlayer`, `getWorld`, `getCommand`, `getItemMeta`,
  `getKiller` can be null. Check before use.
- **Threading**: Bukkit API is main-thread only. Use `BukkitRunnable` for scheduling; use async
  ONLY for file/network I/O and hop back with `runTask` before touching the world.
- **Fail fast and visibly**: validate required config on enable and log a clear `getLogger().warning(...)`
  telling the admin exactly what to fix.
- **Honour the plan's class contracts exactly** (see the LOCKED section in the user message): every
  `new X(...)` call site must match the declared constructor; declared `extends`/`implements` must hold.

## File Formats

- **plugin.yml** (generated for you): `name`, `version`, `main` (fully-qualified main class),
  `api-version` QUOTED (e.g. `api-version: '1.21'`), then `commands:` and `permissions:`. Command
  names are single words; sub-commands are `args[0]`.
- **config.yml** (you generate this when needed): valid YAML, 2-space indent, `#` comments on every
  key, realistic defaults so the plugin works on first run. Booleans `true/false`, lists with `- `.
- **Data files** you create at runtime (e.g. `data.yml`) are separate from config.yml — settings vs
  state. Key player data by UUID string.
- **NamespacedKey**: `new NamespacedKey(plugin, "lower_snake_id")` — lowercase, no spaces; reuse the
  same key instance for read and write.

## Sub-command dispatch (CRITICAL)
Commands are registered in plugin.yml as single words. Sub-commands are passed as args[0].

CORRECT pattern for `/guard spawn|target|move|home`:
```java
@Override
public boolean onCommand(CommandSender sender, Command command, String label, String[] args) {
    if (args.length == 0) { sender.sendMessage("Usage: /guard <spawn|target|move|home>"); return true; }
    switch (args[0].toLowerCase()) {
        case "spawn": return handleSpawn(sender, args);
        case "target": return handleTarget(sender, args);
        case "move":   return handleMove(sender, args);
        case "home":   return handleHome(sender, args);
        default: sender.sendMessage("Unknown sub-command: " + args[0]); return true;
    }
}
```
In onEnable(): `getCommand("guard").setExecutor(this);` — one call, not one per sub-command.
NEVER do `getCommand("guard spawn")` — it will always return null.

## Persistent entity reload
When a plugin saves entity UUIDs to config and reloads on server restart, ALWAYS search the
world for the existing entity by UUID before spawning a new one. Spawning unconditionally creates
duplicate entities every restart:

```java
UUID saved = UUID.fromString(config.getString("entityUUID"));
Entity existing = world.getEntities().stream()
    .filter(e -> e.getUniqueId().equals(saved)).findFirst().orElse(null);
if (existing == null) {
    existing = world.spawn(location, Zombie.class); // only spawn if not found
}
```

## Event listeners
Register every Listener in onEnable() — forgetting this means no events fire:
```java
getServer().getPluginManager().registerEvents(myListener, this);
```
Annotate each handler with `@EventHandler`. Only override `priority` when you specifically need
to run before or after other plugins. Never store a `Player` reference in a field — store the
UUID and resolve it with `Bukkit.getPlayer(uuid)` on use, since players can log out.

## Inventory GUIs
Use a title prefix so your `InventoryClickEvent` handler can recognise your own inventories:
```java
private static final String PREFIX = "MyPlugin: ";

// open
Inventory inv = Bukkit.createInventory(null, 27, PREFIX + "Main Menu");
player.openInventory(inv);

// handle clicks
@EventHandler
public void onInventoryClick(InventoryClickEvent e) {
    if (!e.getView().getTitle().startsWith(PREFIX)) return;
    e.setCancelled(true); // always — prevents item movement and duplication
    // route by full title string, act on e.getSlot() / e.getCurrentItem()
}
```
Inventory size must be a multiple of 9, max 54. Register the listener in onEnable().

## Scheduled tasks
For repeating work use BukkitRunnable, not Java threads:
```java
new BukkitRunnable() {
    @Override public void run() { /* runs on main thread, safe to use Bukkit API */ }
}.runTaskTimer(plugin, 0L, 20L); // delay 0 ticks, repeat every 20 ticks (1 second)
```
Use `runTaskTimerAsynchronously` only for I/O (file/network) — never call Bukkit API from an async task.
Cancel with the returned `BukkitTask` reference or by calling `cancel()` inside the runnable.

## Config management
Call `saveDefaultConfig()` before any `getConfig()` call in `onEnable()`. Always supply a
default so the plugin never throws on a missing key:
```java
int radius = getConfig().getInt("radius", 10);
```
After writing a value, call `saveConfig()` immediately. For a reload command, call
`reloadConfig()` then re-read every cached config value into your fields.

## Reference example (target structure & quality)

A minimal but COMPLETE plugin: main class wires things up, a command class with tab completion,
a config-backed value, and a per-player cooldown. Match this separation and quality.

src/main/java/com/bekololek/generated/HealPlugin.java
```java
package com.bekololek.generated;

import org.bukkit.command.PluginCommand;
import org.bukkit.plugin.java.JavaPlugin;

public class HealPlugin extends JavaPlugin {
    private int cooldownSeconds;

    @Override
    public void onEnable() {
        saveDefaultConfig();
        cooldownSeconds = getConfig().getInt("cooldown-seconds", 30);
        PluginCommand cmd = getCommand("heal");
        if (cmd != null) {
            HealCommand executor = new HealCommand(this);
            cmd.setExecutor(executor);
            cmd.setTabCompleter(executor);
        }
        getLogger().info("HealPlugin enabled.");
    }

    public int getCooldownSeconds() { return cooldownSeconds; }

    public void reloadValues() { reloadConfig(); cooldownSeconds = getConfig().getInt("cooldown-seconds", 30); }
}
```

src/main/java/com/bekololek/generated/HealCommand.java
```java
package com.bekololek.generated;

import org.bukkit.command.Command;
import org.bukkit.command.CommandExecutor;
import org.bukkit.command.CommandSender;
import org.bukkit.command.TabCompleter;
import org.bukkit.entity.Player;

import java.util.Collections;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import java.util.UUID;

public class HealCommand implements CommandExecutor, TabCompleter {
    private final HealPlugin plugin;
    private final Map<UUID, Long> cooldownUntil = new HashMap<>();

    public HealCommand(HealPlugin plugin) { this.plugin = plugin; }

    @Override
    public boolean onCommand(CommandSender sender, Command command, String label, String[] args) {
        if (args.length > 0 && args[0].equalsIgnoreCase("reload")) {
            if (!sender.hasPermission("heal.reload")) { sender.sendMessage("§cNo permission."); return true; }
            plugin.reloadValues();
            sender.sendMessage("§aReloaded.");
            return true;
        }
        if (!(sender instanceof Player player)) { sender.sendMessage("Players only."); return true; }
        long now = System.currentTimeMillis();
        Long until = cooldownUntil.get(player.getUniqueId());
        if (until != null && until > now) {
            sender.sendMessage("§cWait " + ((until - now) / 1000) + "s.");
            return true;
        }
        player.setHealth(player.getAttribute(org.bukkit.attribute.Attribute.GENERIC_MAX_HEALTH).getValue());
        cooldownUntil.put(player.getUniqueId(), now + plugin.getCooldownSeconds() * 1000L);
        player.sendMessage("§aHealed!");
        return true;
    }

    @Override
    public List<String> onTabComplete(CommandSender sender, Command command, String alias, String[] args) {
        if (args.length == 1 && sender.hasPermission("heal.reload")) return List.of("reload");
        return Collections.emptyList();
    }
}
```

src/main/resources/config.yml
```yaml

```

This is exactly the shape and care expected: separated classes, null-checked `getCommand`, cached
